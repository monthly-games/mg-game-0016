## 1. 재미 요소 (Fun Factors)

### 1.1 핵심 재미 (Core Fun)
| 요소 | 설명 | 강도 |
|------|------|------|
| **덱 구성** | 30장의 카드로 전략적 덱 구성 | ★★★★★ |
| **카드 수집** | 다양한 카드 수집과 강화 | ★★★★★ |
| **스토리** | JRPG 스타일의 스토리 진행 | ★★★★☆ |
| **전투 전략** | 덱 구성에 따른 오토배틀 결과 | ★★★★☆ |

### 1.2 만족감 요소 (Satisfaction)
- **희귀 카드**: 레전더리 카드 획득
- **덱 완성**: 시너지 덱 완성의 성취감
- **스토리 진행**: 챕터 클리어 시 스토리 해금
- **히어로 해금**: 새로운 플레이어블 히어로 획득

### 1.3 도전 요소 (Challenge)
```
도전 구조:
├── 카드 코스트 관리 (마나 시스템)
├── 카드 시너지 조합
├── 챕터별 보스 전략
├── 특수 조건 스테이지
└── PvP 덱 경쟁
```

### 1.4 사회적 요소 (Social)
- **덱 공유**: 인기 덱 레시피 공유
- **PvP**: 비동기 PvP 대전
- **길드**: 길드 레이드
- **리더보드**: 챕터/PvP 랭킹

## 2. 게임 루프 (Game Loop)

### 2.1 코어 루프 (3~5분)
```
┌─────────────────────────────────────────────────┐
│                    코어 루프                      │
├─────────────────────────────────────────────────┤
│  [덱 확인] → [스테이지 선택] → [오토배틀]         │
│       ↑                            ↓             │
│       └──── [보상 획득] ← [스테이지 클리어] ←┘   │
└─────────────────────────────────────────────────┘
```

**단계별 상세:**
1. **덱 확인** (10초): 현재 덱 구성 확인
2. **스테이지 선택** (5초): 진행할 스테이지 선택
3. **오토배틀** (60~180초): 덱 기반 자동 전투
4. **스테이지 클리어** (5초): 클리어 결과 확인
5. **보상 획득** (5초): 카드, 골드, 경험치 획득

### 2.2 세션 루프 (10~15분)
```
[접속] → [일일 보상] → [퀘스트 확인] → [스토리 진행] → [카드 관리]
                                            ↓
                                [덱 조정] ← [카드 획득]
```

### 2.3 메타 루프 (장기)
```
┌──────────────────────────────────────────────────────┐
│                     메타 진행                         │
├──────────────────────────────────────────────────────┤
│  [스테이지 클리어] → [카드 획득] → [덱 강화]          │
│         ↓                              ↓             │
│  [새 챕터 해금]                  [히어로 해금]        │
│         ↓                              ↓             │
│  [더 어려운 콘텐츠] ←──────────────────┘             │
└──────────────────────────────────────────────────────┘
```

### 2.4 플로우 다이어그램
```
[앱 시작]
    ↓
[메인 화면] ─────→ [카드 도감]
    │              [덱 편집]
    │              [상점]
    │              [설정]
    ↓
[스토리] → [챕터 선택] → [스테이지] → [배틀] → [결과]
```

## 3. BM 모델 (Business Model)

### 3.1 수익화 전략
| 채널 | 비중 | 설명 |
|------|------|------|
| **카드 팩** | 40% | 가챠 시스템 |
| **히어로 번들** | 25% | 히어로 해금 팩 |
| **리워드 광고** | 20% | 추가 드롭, 보상 2배 |
| **편의 기능** | 15% | 덱 슬롯, 에너지 |

### 3.2 과금 포인트
```
비과금 유저 경험:
├── 모든 스토리 클리어 가능
├── 무료 일일 뽑기 제공
├── 기본 히어로 2종 제공
└── 카드 분해로 제작 가능

과금 유저 혜택:
├── 프리미엄 카드 팩 ($0.99~$49.99)
├── 히어로 번들 ($4.99~$14.99)
├── 크리스탈 패키지 ($0.99~$99.99)
├── 스타터 팩 ($4.99)
└── 배틀 패스 ($9.99/시즌)
```

### 3.3 가격 정책
| 상품 | 가격 | 가치 |
|------|------|------|
| 일반 팩 (5장) | $0.99 | R 이상 확정 |
| 고급 팩 (10장) | $4.99 | SR 1장 확정 |
| 프리미엄 팩 (10장) | $9.99 | 레전더리 1장 확정 |
| 히어로 번들 | $4.99~14.99 | 히어로 + 전용 카드 |
| 스타터 팩 | $4.99 | SR 히어로 + 팩 5개 |

### 3.4 리텐션 메커니즘
- **일일 보상**: 로그인 무료 팩
- **일일 퀘스트**: 전투 N회, 카드 강화
- **스토리 진행**: 챕터별 스토리 해금
- **시즌 패스**: 60일 시즌 보상

## 4. 레벨 디자인 (Level Design)

### 4.1 난이도 곡선
```
적 강도
  ▲
  │                              ╭──────
  │                         ╭────╯
  │                    ╭────╯    <- 보스
  │               ╭────╯
  │          ╭────╯
  │     ╭────╯
  │╭────╯
  └──────────────────────────────────────→ 챕터
    1    2    3    4    5    6    7
```

**난이도 공식:**
```
적_전투력 = 기본 × (1 + 챕터 × 0.15)
권장_덱파워 = 1000 + (챕터 × 500)
```

### 4.2 콘텐츠 진행
| 챕터 | 해금 조건 | 새로운 요소 |
|------|----------|-------------|
| 1 | 시작 | 기본 카드 50종, 튜토리얼 |
| 2 | 챕터1 클리어 | 중급 카드, 시너지 해금 |
| 3 | 챕터2 클리어 | 고급 카드, PvP 해금 |
| 4 | 챕터3 클리어 | 희귀 카드, 길드 해금 |
| 5 | 챕터4 클리어 | 레전더리 드롭 |
| 6 | 챕터5 클리어 | 엔드게임 덱 |
| 7 | 챕터6 클리어 | 무한 던전 |

### 4.3 밸런싱 공식
```dart
// 덱 파워 계산
int calculateDeckPower(List<Card> deck) {
  return deck.map((c) =>
    c.attack + c.defense + c.effect.power
  ).sum.toInt() + getSynergyBonus(deck);
}

// 카드 효과 계산
int calculateCardEffect(Card card, BattleState state) {
  int base = card.effect.baseValue;
  double levelBonus = 1 + card.level * 0.1;
  double synergyBonus = state.activeSynergies.contains(card.type) ? 1.3 : 1.0;
  return (base * levelBonus * synergyBonus).toInt();
}

// 카드 강화 비용
int upgradeCost(Card card) {
  return card.rarity.baseCost * pow(1.5, card.level).toInt();
}
```

### 4.4 보상 체계
| 보상 유형 | 획득 조건 | 수량 |
|-----------|----------|------|
| 스테이지 골드 | 클리어 | 50~200 |
| 카드 드롭 | 클리어 (확률) | C 70%, R 25%, SR 5% |
| 챕터 보상 | 챕터 완료 | 카드 팩 + 크리스탈 |
| 일일 보상 | 출석 | 무료 팩 |
| 카드 파편 | 카드 분해 | 등급별 상이 |

## 5. UI/UX 설계

### 5.1 정보 계층
```
레벨 1 (항상 표시):
├── 골드/크리스탈
├── 에너지
├── 덱 파워
└── 알림 배지

레벨 2 (상호작용 시):
├── 카드 상세 정보
├── 덱 편성 화면
├── 전투 진행
└── 시너지 정보

레벨 3 (팝업):
├── 팩 오픈 연출
├── 카드 획득
├── 결과 화면
└── 설정 메뉴
```

### 5.2 네비게이션 흐름
```
[메인 화면]
    ├── [스토리] → [챕터] → [스테이지] → [배틀]
    ├── [덱] → [덱 목록] → [편집]
    ├── [카드] → [카드 목록] → [강화/분해]
    ├── [뽑기] → [팩 선택] → [결과]
    ├── [상점] → [상품 목록]
    └── [설정] → [옵션]
```

### 5.3 피드백 시스템
| 이벤트 | 시각 | 청각 | 촉각 |
|--------|------|------|------|
| 카드 사용 | 카드 이펙트 | 카드음 | 진동 (약) |
| 시너지 발동 | 시너지 연출 | 시너지음 | 진동 (중) |
| 승리 | 승리 연출 | 승리 BGM | 진동 (중) |
| 레전더리 획득 | 황금 연출 | 팡파레 | 진동 (강) |

### 5.4 접근성
- **자동 전투**: 기본 자동 진행
- **배속 기능**: 1x, 2x, 3x
- **덱 추천**: AI 덱 추천
- **색맹 모드**: 카드 타입 패턴
- **폰트 크기**: 3단계 조절
